# Contributing to PingUp 🟢

First off, thank you for considering contributing to PingUp! It's people like you that make PingUp such a great tool for everyone.

### 🚀 Getting Started

1. **Fork the Repository**: Create your own copy of the project.
2. **Clone the Repository**: 
   ```bash
   git clone https://github.com/sabeenaviklar/PingUp.git
   ```
3. **Create a Branch**: 
   ```bash
   git checkout -b feature/YourFeatureName
   ```
4. **Make Your Changes**: Implement your feature or fix.
5. **Commit Your Changes**: 
   ```bash
   git commit -m 'Add some feature'
   ```
6. **Push to the Branch**: 
   ```bash
   git push origin feature/YourFeatureName
   ```
7. **Open a Pull Request**: Submit your changes for review.

### 📋 Contribution Guidelines

- **Code Quality**: Ensure your code follows the existing style and is well-documented.
- **Issue First**: Please open an issue first to discuss what you would like to change.
- **Commit Messages**: Use clear and descriptive commit messages.
- **Testing**: Test your changes thoroughly before submitting.

### 🛠️ Git Cheat Sheet for Beginners

If you are new to Git, here are the most common commands you'll need:

- **Check status**: `git status` (See what files you've changed)
- **Add changes**: `git add .` (Prepare all your changes for a commit)
- **Commit**: `git commit -m "Brief description of change"`
- **Sync with main**: `git pull origin main` (Keep your local code up to date)

### For working on any issue:
To ensure a smooth workflow and respect everyone's time, please follow these steps before writing any code:

1.Find or Create an Issue: If you see an open issue, leave a comment asking to be assigned. If you have a new idea or found a bug, create a new issue first and ask to work on it!
2.Wait for Approval: Wait for a maintainer to approve the issue and officially assign you to it.
3.Submit your PR: Once assigned, you can start coding and submit your Pull Request.
Note: Please do not submit Pull Requests for issues that have not been assigned to you
